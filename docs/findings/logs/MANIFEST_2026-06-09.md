## TL;DR of the arc
1. The reflective loop is a **rank-independent moat** at the token-PAIR level — RIGID even
   against a genuinely orthogonal deterministic pair (dim 256, cos +0.018). (Confirms +
   extends CC's 2026-06-09 migration finding; independent cross-replication, seed 1 identical.)
2. A **Fix 2 governor** (gnov W=10 / T=0.65 / ≥2 sources; attenuate loop gain 1.0→0.5,
   floor 0.45, rails = manip + attractor-count) was built and validated **on a MOCK
   orthogonal-B regime**: recovers ~28× plasticity (migration +0.006→+0.166), benign/attack
   correctly don't loosen. BUT it woke the manip layer ~14% at gain 0.5.
3. **The live-generator run overturned the mock story.** On the REAL generator with real
   injected tokens, regime *distributions* do NOT separate (means collinear ~0.89) because
   the generator emits a dominant **common-mode** direction → the std trigger is **dormant**
   at dim 64 AND 256. A **common-mode-removed trigger** engages it (94% loosen, no retraining)
   but migration recovery is **small (+0.024)** and the manip cost is **~1–4% at gain 0.5,
   0% at gain 0.6**. So the mock's 28× recovery AND 14% manip were **perfect-orthogonality
   artifacts**.
4. **dim is NOT the lever** — confirmed across 64/256/512: common-mode dilutes too slowly
   (regime-mean sep +0.96→+0.89→+0.85, still collinear), and the common-mode-trigger
   migration recovery is **flat (+0.024→+0.027)** despite doubling dim. **The lever is
   training the generator** to put regime differences in high-energy separable directions.

**Recommendation on file:** demote Fix 2's priority; if built, its trigger must project out
the common-mode (operating point gain ≈0.6, 0% manip); point the real work at the generator.
